Specification
Using ID ID is the strong specifier, if you ever need to change a single element exactly, use ID. Unlike Class it can only be used for 1 element per document #"idname" (style: detail;)
Classes The second priority in specification, classes can be used to target multiple elements throught a document .classname (sytle:detail) You can also layer and stack classes Ex. Class= Green bold CSS- .green and .bold can have different rules that will both apply or not apply to class=red bold
Psuedo Classes (think attribute then actions) Syntax #idname:Psuedo (color:green;) Different types of actions: focus, visited, disabled, active, hover
Class-Element/ Element-Class If you are wanting to add a style to multiple elements, then we can use these syntax. Class-element will only apply the style to (h1) inside of your class, where Element class can apply to multiple classes throughout the document. Not really sure what situation would warrant using one or the other

The Box Model
Resetting Defaults Major web browsers have a default margins stylesheet for any info that doesn't already have one. Set all (*) margins and padding to 0 to make sure you are starting with a clean slate Syntax= *(margin:0; padding:0;)
New box method Use the new box method styling to make sure your content margins adjust to your styling Syntax: *(box-sizing:border-box;)
Content The height & width style applies to the content area Syntax for H&W. element (height:100px;)
Padding The amount of area between the Content and the border Syntax: Either use (padding-top/right/bottom/left) or CSS reads (Padding: 5px 10px 7px 10px) clockwise starting at the top
Border The thickness and style of the border Syntax. Border: (px or thin, medium, thick) (style) (color);
Border Radius Can add curvurture to your borders Syntax= Border-radius:(px or 50% for circle)
Margin The amount of area between the border and the outside edge of the element Syntax: Either use (margin-top/right/bottom/left) or CSS reads (Padding: 5px 10px 7px 10px) clockwise starting at the top
Auto Margins Center an element to fit its container. However, you must set a width so that your content doesn't just take the width of the screen. Syntax= width:400px (400px is used commonly because its smaller than most screens) + (margin: 0 auto)
Min and Max H&W With the vast differences in screen size, we need to put min and maxs for H&W to make adjust the UI for different screen sizes. Syntax= (min-height: 200px;) or (max-width:400px;)
Over flow Over flow determines what happens to content that overflows their own H&W. Different options include (hidden, scroll, or visible) Syntax= (overflor:scroll;)
Visibility For Works in progress, we can use visibility to save a spot for future elements, without it being accessible to users (hidden, visible, collapse) Syntax= (visibility: hidden;)

Display and Positioning
Position Telling the page where you want your block located. 1. Static- Exactly where it is, default. 2. Relative must be accompanied by a top, left, bottom px #. 3.Absolute-it won't move despite scroll. 4. Fixed-will stay in position with scroll (think headers and nav bars). 5.Sticky- Will scroll until you reach the bottom of the parent container.
Z-Index Think of this as the "bring forward" or "send to back" rating. The higher the number, the higher the priority since all are 0 Syntax= z-index: 1;
Float Used to tell an element to go as far right or left as possible on a page Syntax= (float:right/left)
Display Can change the position of the blocks in a document. 1. Inline- keeps an element in line 2. Block- creates a seperate block for your info 3. Inline-block- Creates blocks in line Syntax= (display: inline-block;)
Clear Clear is a function to add with float. It tells the block what to do if there are two floating elements bumping
